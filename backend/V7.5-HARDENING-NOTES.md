# JobNova V7.5 hardening notes

This version adds the durable production backbone layer:

- Postgres migration for `background_jobs`
- BullMQ + Redis queue support with local fallback
- async job endpoints under `/jobs/*`
- frontend background job monitoring screens
- request-safe job persistence through a repository layer

## Queue modes

### Redis present
When `REDIS_URL` is configured, BullMQ is used for durable queueing with retries and exponential backoff.

### Redis absent
The app falls back to a local async queue so development still works without external infrastructure.

## Migration

Run `001_v75_background_jobs.sql` after setting `DATABASE_URL`.

## Frontend UX

New hidden routes:
- `/(app)/jobs`
- `/(app)/jobs/[id]`

These provide progress tracking for ATS, exports, and interview scoring.
