# JobNova V7.6 Autopilot Hardening Notes

This pack adds the first persistent approval workflow for recruiter autopilot packages.

## Added
- Postgres migration for `application_packages`
- repository layer with local JSON fallback
- package persistence from completed `application-package` background jobs
- approval endpoint
- retry-export endpoint that requeues a durable export job
- frontend package review screen
- job detail CTA to save/open completed application packages

## New backend endpoints
- `GET /jobs/autopilot/packages`
- `POST /jobs/autopilot/packages/from-job/:jobId`
- `GET /jobs/autopilot/packages/:packageId`
- `POST /jobs/autopilot/packages/:packageId/approve`
- `POST /jobs/autopilot/packages/:packageId/retry-export`

## Important truth
This is a stronger workflow layer, but it is still not a fully human-reviewed recruiter submission engine. The package generation is still driven by the existing ATS/rewrite service and local fallback persistence remains available when Postgres is not configured.
